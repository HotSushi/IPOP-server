IP over P2P Web Interface

What is IPOP?

IPOP (IP-over-P2P) is an open-source user-centric software virtual network allowing end users to define and create their own virtual private networks (VPNs). IPOP virtual networks provide end-to-end tunneling of IP or Ethernet over “TinCan” links setup and managed through a control API to create various software-defined VPN overlays.

Uses of the Web Interface

The web interface helps you create VPNs with GUI. It currently supports two (GVPN, AdminGVPN) out of the three VPN types IPOP has. It provides you with visualization tools to monitor your VPN easily. The Web-UI also supports integration with Ganglia monitoring tool. You can also see log files that were sent by ejabberd servers and client nodes.

VPN Types

  • GVPN: GroupVPN, is the basic version of VPN where the user is expected to use JIDs which are already registered in xmpp server and relationships with other JIDs are already established.

  • ADMIN-GVPN: Admin-GroupVPN, uses the functionality of MUC rooms, hence avoiding the trouble of adding relationships. Admin of the room can add nodes, remove nodes. Here too, JIDs are assumed to be valid.

  • BATCH-GVPN: Batch-GroupVPN, is similar to GVPN but with added functionality of automatically generating JIDs, adding relationships, removing JIDs, which is useful when there are large number of node.

Multiple VPNs

This web interface helps you in managing multiple IPOP VPNs at the same time.

You can change IP, stop nodes or delete nodes from the user interface.

This server is expected to run on a static IP so that client nodes can connect to it directly.

The server connects to the client nodes through the admin JID provided for each VPN.

Whenever a stop/change IP button is clicked, the server switches to the respective admin JID and sends msg to the client.

Graphs

Uses D3.js for displaying graphs

Clicking on the nodes takes you to the relevant data, for ex. Monitoring info or Logs for that node

Hovering over the nodes provides info such as its JID, Password, IP

Color of the nodes reflects whether the node is running

Admin GVPN

The WebUI supports AdminGVPN Github link

AdminGVPN uses MUC rooms to invite member nodes or remove nodes, which saves trouble of adding relationships between them.

This server preserves environment for each of the existing AdminGVPNs locally {ProjectDir}/static/scripts/db.

WebUI also supports removing nodes from rooms.

The wrapper built around admingvpn controller supports inviting new members too, but this functionality is not used yet.

Batch GVPN

Behaves similar to normal GVPN, but with additional functionality of automatic creation of nodes and adding friend relationship. Doing this manually for large number of node will be infeasible

Uses Groupvpn-config for generating instuctions for creating nodes and establishing relations.

Captures IP settings provided by Groupvpn-config

Requires Ejabberd-IPOP Interface server to be running on the ejabberd server. (Port: 7000)

Server sends ejabberd instructions and sudo password to this Ejabberd-IPOP interface which then executes it locally.

Client Interface

The Client-UI eases the tasks of launching IPOP and controller.

It requires Server's address:port and Client Node's JID for logging in.

Client-UI automatically fetches config data from the server, generates config files and launches respective type of the controller.

Client-UI fetches config file securely by registering it's public key with the server, and getting the encrypted config data.

It also does the task of periodically uploading log files to the server.

Logs

Server provides interface for uploading logs.

Client-UI uploads logs when it starts, untill it is stopped.

A script is written for uploading ejabberd logs to the server, but it has to be cron-scheduled

Logs of a particular node can be seen by clicking on it on the graph.

Logs can be flushed by clicking on the button provided on the interface

Ganglia Monitoring

IPOP is already integrated with Ganglia Monitoring Tool .

The web-ui collects graphs from the ganglia web interface.

Monitoring info is only available for clients nodes and not for xmpp servers.

Monitoring info can be accessed by clicking on the node in the graph.

Time period of the graph can be changed.

Small Screen Support

The UI is designed with bootstrap.

Most of the features can be used with smaller screen.

If the server has a static IP it is possible to use the features through mobile device.

The interface is not very attractive for smaller screens, but it can be worked upon.
